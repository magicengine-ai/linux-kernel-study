## 📊 内存布局

### x86_64 内存布局

```
用户空间 (0x0000000000000000 - 0x00007FFFFFFFFFFF)
├── 代码段 (.text)
├── 数据段 (.data)
├── BSS 段 (.bss)
├── 堆 (Heap) - 向上增长
├── 内存映射区 (mmap)
└── 栈 (Stack) - 向下增长

内核空间 (0xFFFF800000000000 - 0xFFFFFFFFFFFFFFFF)
├── 直接映射区 (Direct Map) - 物理内存线性映射
├── vmalloc 区 - 虚拟连续内存
├── 模块区 (Modules) - 内核模块
└── 内核代码/数据
```

### 内存区域 (Zones)

```
ZONE_DMA      - DMA 设备可用内存 (0-16MB)
ZONE_DMA32    - 32 位 DMA 内存 (16MB-4GB)
ZONE_NORMAL   - 正常内存 (4GB-896MB 32 位)
ZONE_HIGHMEM  - 高端内存 (>896MB 32 位)
```

## 📦 页面分配器

### 基本概念

- **页面 (Page)**: 内存管理基本单位，通常 4KB
- **页框 (Page Frame)**: 物理内存页面
- **描述符 (struct page)**: 页面元数据

### struct page 结构

```c
struct page {
    unsigned long flags;        // 页面标志
    struct list_head lru;       // LRU 链表
    struct address_space *mapping;  // 映射地址空间
    pgoff_t index;              // 页面索引
    void *private;              // 私有数据
    refcount_t _refcount;       // 引用计数
    // ... 更多字段
};
```

### 页面标志

```c
#define PG_locked          0    // 页面锁定
#define PG_referenced      11   // 页面被引用
#define PG_dirty           13   // 页面脏
#define PG_writeback       18   // 正在回写
#define PG_slab            9    // SLAB 分配器
#define PG_head            16   // 大页头页
#define PG_tail            17   // 大页尾页
```

### 页面分配 API

```c
// 分配单页
struct page *alloc_page(gfp_t mask);

// 分配连续页
struct page *alloc_pages(gfp_t mask, unsigned int order);

// 释放页面
void __free_page(struct page *page);
void __free_pages(struct page *page, unsigned int order);

// 虚拟地址分配
void *get_zeroed_page(gfp_t mask);
unsigned long get_free_page(gfp_t mask);

// 释放虚拟地址
void free_page(unsigned long addr);
```

### GFP 标志

```c
// 行为标志
GFP_KERNEL      // 标准内核分配 (可睡眠)
GFP_ATOMIC      // 原子分配 (不可睡眠)
GFP_NOWAIT      // 不等待
GFP_NOIO        // 不允许 I/O
GFP_NOFS        // 不允许文件系统操作

// 区域标志
__GFP_DMA       // DMA 区域
__GFP_DMA32     // DMA32 区域
__GFP_HIGHMEM   // 高端内存

// 修改标志
__GFP_ZERO      // 清零页面
__GFP_REPEAT    // 重试分配
__GFP_NORETRY   // 不重试
```

## 🔧 内核内存分配器

### 三种分配器对比

| 特性 | SLAB | SLUB | SLOB |
|------|------|------|------|
| 设计目标 | 性能 | 简单高效 | 嵌入式 |
| 内存开销 | 中 | 低 | 最低 |
| 碎片化 | 低 | 低 | 中 |
| 适用场景 | 通用 | 默认 | 嵌入式/小内存 |
| 调试支持 | 强 | 中 | 弱 |

### SLAB 分配器

```
SLAB 架构:
├── Cache (缓存)
│   ├── kmem_cache
│   └── 对象大小固定
├── Slab (片)
│   ├── 连续页面
│   └── 包含多个对象
└── Object (对象)
    └── 实际数据结构
```

```c
// 创建缓存
struct kmem_cache *kmem_cache_create(
    const char *name,
    unsigned int size,
    unsigned int align,
    slab_flags_t flags,
    void (*ctor)(void *)
);

// 分配对象
void *kmem_cache_alloc(struct kmem_cache *cachep, gfp_t flags);

// 释放对象
void kmem_cache_free(struct kmem_cache *cachep, void *objp);

// 销毁缓存
void kmem_cache_destroy(struct kmem_cache *cachep);
```

### SLUB 分配器 (默认)

```
SLUB 架构:
├── Kmem Cache
│   ├── per-CPU Slabs (CPU 本地)
│   ├── Partial Slabs (部分使用)
│   ├── Full Slabs (已满)
│   └── Free Slabs (空闲)
└── Objects
```

```c
// SLUB 使用与 SLAB 相同 API
// 内部自动选择

// 查看 SLUB 信息
cat /sys/kernel/slab/*/objects      // 对象数
cat /sys/kernel/slab/*/objsize      // 对象大小
cat /sys/kernel/slab/*/slabs        // Slab 数
```

### SLOB 分配器

```
SLOB 架构:
├── 简单链表
└── 首次适配算法

适用于：
- 嵌入式系统
- 内存 < 32MB
- 简单场景
```

## 🔄 页面回收

### 回收机制

```
页面回收流程:
1. kswapd 内核线程唤醒
2. 扫描 LRU 链表
3. 选择牺牲页面
4. 写回脏页 (如果需要)
5. 释放页面
```

### LRU 链表

```
LRU 链表:
├── LRU_ACTIVE_ANON     // 活跃匿名页
├── LRU_ACTIVE_FILE     // 活跃文件页
├── LRU_INACTIVE_ANON   // 非活跃匿名页
├── LRU_INACTIVE_FILE   // 非活跃文件页
├── LRU_UNEVICTABLE     // 不可回收页
└── LRU_LISTS           // 链表数量
```

### 回收 API

```c
// 页面回收
unsigned long try_to_free_pages(struct zonelist *zonelist,
                                gfp_t gfp_mask,
                                nodemask_t *nodemask);

// 写回脏页
int write_one_page(struct page *page);

// 收缩缓存
void shrink_all_memory(int nr_to_scan);
```

## 🚨 OOM 杀手

### OOM 机制

```
OOM 触发条件:
1. 内存耗尽
2. 无法回收内存
3. 分配失败

OOM 选择算法:
- 计算 oom_score
- 选择得分最高的进程
- 杀死进程释放内存
```

### OOM 分数

```bash
# 查看 OOM 分数
cat /proc/[pid]/oom_score
cat /proc/[pid]/oom_score_adj

# 调整 OOM 分数
echo -1000 > /proc/[pid]/oom_score_adj  # 保护进程
echo 1000 > /proc/[pid]/oom_score_adj   # 优先杀死
```

### OOM 代码

```c
// OOM 杀手入口
void out_of_memory(struct zonelist *zonelist, gfp_t gfp_mask,
                   int order, nodemask_t *nodemask,
                   bool force_kill);

// 选择牺牲进程
struct task_struct *select_bad_process(unsigned long *ppoints,
                                       struct oom_control *oc);
```

## 🎯 实践练习

### 练习 1: 内核模块内存分配

```c
// mem_alloc.c
#include <linux/module.h>
#include <linux/slab.h>
#include <linux/vmalloc.h>

static char *kmem_ptr;
static char *vmem_ptr;
static struct kmem_cache *cache;

static int __init mem_init(void)
{
    // kmalloc (物理连续)
    kmem_ptr = kmalloc(1024, GFP_KERNEL);
    if (!kmem_ptr)
        return -ENOMEM;
    
    // vmalloc (虚拟连续)
    vmem_ptr = vmalloc(1024 * 1024);
    if (!vmem_ptr) {
        kfree(kmem_ptr);
        return -ENOMEM;
    }
    
    // kmem_cache
    cache = kmem_cache_create("my_cache", 64, 0, 0, NULL);
    if (!cache) {
        kfree(kmem_ptr);
        vfree(vmem_ptr);
        return -ENOMEM;
    }
    
    pr_info("Memory allocation successful\n");
    return 0;
}

static void __exit mem_exit(void)
{
    kmem_cache_destroy(cache);
    kfree(kmem_ptr);
    vfree(vmem_ptr);
    pr_info("Memory freed\n");
}

module_init(mem_init);
module_exit(mem_exit);
MODULE_LICENSE("GPL");
```

## ✅ 总结

Linux 内存管理核心要点：

- **页面分配器** - 基础内存分配单位
- **SLUB 分配器** - 默认内核对象分配器
- **虚拟内存** - VMA 管理进程地址空间
- **页面回收** - LRU 算法 + kswapd
- **KASAN/KFENCE** - 内存错误检测
- **OOM 杀手** - 内存耗尽保护机制
- **cgroup** - 内存资源控制

内存管理是内核最复杂的子系统之一！
