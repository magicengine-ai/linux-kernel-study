# 02-Linux 内核构建系统详解

> 版本：v7.0-rc4  
> 更新时间：2026-03-20

---

## 📋 概述

Linux 内核构建系统是一个复杂的 Kbuild 系统，基于 GNU Make 构建。

### 核心文件

| 文件 | 大小 | 作用 |
|------|------|------|
| `Makefile` | 74KB | 主构建脚本 |
| `Kbuild` | 2.8KB | 构建系统配置 |
| `Kconfig` | 582B | 配置选项定义 |

---

## 🏗️ 构建系统架构

```
顶层 Makefile
    │
    ├── Kbuild          # 构建规则
    ├── Kconfig         # 配置选项
    │
    ├── arch/           # 架构特定 Makefile
    ├── drivers/        # 驱动 Makefile
    ├── fs/             # 文件系统 Makefile
    ├── kernel/         # 核心 Makefile
    └── ...             # 各子系统 Makefile
```

## 🔧 依赖要求

### 最低要求
- **GNU Make**: >= 4.0
- **编译器**: GCC 或 Clang
- **binutils**: 链接器工具
- **libssl-dev**: OpenSSL 开发库

### 可选依赖
- **Rust**: 1.70+ (用于 Rust 内核模块)
- **pahole**: BTF 调试信息
- **libelf**: ELF 文件处理
- **bc**: 计算器 (Kconfig 使用)

## 📦 内核配置

### 配置方式

```bash
# 1. 交互式配置 (推荐)
make menuconfig

# 2. 图形界面配置
make xconfig        # Qt 界面
make gconfig        # GTK 界面
make nconfig        # ncurses 界面

# 3. 基于现有配置
make oldconfig      # 基于 .config
make defconfig      # 默认配置
make allyesconfig   # 全选 Y
make allnoconfig    # 全选 N

# 4. 随机配置 (测试用)
make randconfig

# 5. 使用特定配置
make <defconfig>    # 如：make x86_64_defconfig
```

### 配置文件

| 文件 | 说明 |
|------|------|
| `.config` | 当前配置 (生成) |
| `arch/x86/configs/` | x86 架构预设配置 |
| `kernel/configs/` | 通用预设配置 |

### 配置选项格式

```kconfig
# Kconfig 语法示例
config EXT4_FS
    tristate "Ext4 filesystem support"
    depends on BLOCK
    select CRC32
    select JBD2
    help
      This is the extended version of ext3 filesystem.
      Say Y here if you want to use ext4.
```

**选项类型：**
- `bool` - 是/否
- `tristate` - 内置/模块/否 (Y/M/N)
- `int` - 整数
- `hex` - 十六进制
- `string` - 字符串

## 🔨 编译流程

### 标准编译

```bash
# 1. 配置内核
make menuconfig

# 2. 编译内核和模块
make -j$(nproc)

# 3. 安装模块
make modules_install

# 4. 安装内核
make install
```

### 编译输出

```
输出目录/
├── arch/x86/boot/bzImage      # 压缩内核镜像
├── vmlinux                    # 未压缩内核 (调试用)
├── System.map                 # 内核符号表
├── .config                    # 配置文件
├── Module.symvers             # 模块符号版本
└── modules/                   # 内核模块目录
```

## 📦 模块编译

### 内置模块 vs 外部模块

#### 内置模块编译
```bash
# 在内核源码树内
make M=drivers/net
```

#### 外部模块编译
```bash
# 独立模块目录
make -C /lib/modules/$(uname -r)/build M=$(pwd) modules
```

### 模块 Makefile 示例

```makefile
# Makefile
obj-m += mymodule.o
mymodule-y := main.o utils.o

all:
	make -C /lib/modules/$(uname -r)/build M=$(PWD) modules

clean:
	make -C /lib/modules/$(uname -r)/build M=$(PWD) clean
```

## 📁 Kbuild 语法

### 目标定义

```makefile
# 简单目标
obj-y += foo.o              # 内置
obj-m += bar.o              # 模块

# 复合目标
obj-y += net/
obj-y += drivers/

# 条件编译
obj-$(CONFIG_EXT4_FS) += ext4/
obj-$(CONFIG_NET) += net/

# 多文件模块
obj-m += mymod.o
mymod-y := file1.o file2.o
mymod-$(CONFIG_FEATURE) += feature.o
```

### 目录遍历

```makefile
# 进入子目录编译
obj-y += subdir/

# 子目录 Makefile
# subdir/Makefile
obj-y += foo.o bar.o
```

## 🎯 实践练习

### 练习 1: 编译最小内核

```bash
# 1. 最小配置
make allnoconfig
echo "CONFIG_64BIT=y" >> .config
echo "CONFIG_X86_64=y" >> .config
make olddefconfig

# 2. 编译
make -j$(nproc)

# 3. 测试
qemu-system-x86_64 -kernel arch/x86/boot/bzImage
```

### 练习 2: 编写 Hello World 模块

```c
// hello.c
#include <linux/module.h>
#include <linux/kernel.h>

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("Hello World Module");

static int __init hello_init(void)
{
    pr_info("Hello, Kernel!\n");
    return 0;
}

static void __exit hello_exit(void)
{
    pr_info("Goodbye, Kernel!\n");
}

module_init(hello_init);
module_exit(hello_exit);
```

```makefile
# Makefile
obj-m += hello.o

all:
	make -C /lib/modules/$(uname -r)/build M=$(PWD) modules

clean:
	make -C /lib/modules/$(uname -r)/build M=$(PWD) clean
```

```bash
# 编译和加载
make
sudo insmod hello.ko
dmesg | tail
sudo rmmod hello
```

## ✅ 总结

Linux 内核构建系统特点：

- **Kbuild 系统** - 基于 Make 的复杂构建系统
- **Kconfig 配置** - 灵活的配置选项管理
- **模块化设计** - 支持内置和模块两种方式
- **跨平台支持** - 23 种架构统一构建接口
- **Rust 支持** - 新增 Rust 编译工具链

掌握构建系统是内核开发的第一步！
