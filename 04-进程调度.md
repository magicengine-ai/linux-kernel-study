## 📋 概述

Linux 内核调度器负责：
- 决定哪个进程在 CPU 上运行
- 进程优先级管理
- 负载均衡
- 实时任务支持
- 能效优化

## 🎯 进程优先级

### 优先级范围

```
实时优先级 (RT):
├── 99 (最高)
├── 98
├── ...
└── 1 (最低)

普通优先级 (Nice):
├── -20 (最高优先级)
├── 0 (默认)
└── 19 (最低优先级)

idle 优先级:
└── 120 (最低，空闲任务)
```

### priority 到 nice 映射

```c
// kernel/sched/prio.h
#define MAX_RT_PRIO     100
#define MAX_PRIO        (MAX_RT_PRIO + 40)
#define DEFAULT_PRIO    (MAX_RT_PRIO + 20)

// 优先级转换
static inline int rt_prio(int prio)
{
    if (unlikely(prio < MAX_RT_PRIO))
        return 1;
    return 0;
}

static inline int task_prio(const struct task_struct *p)
{
    return p->prio - MAX_RT_PRIO;
}
```

## ⚖️ CFS 完全公平调度器

### 核心概念

```
CFS 设计目标:
├── 公平性 - 每个任务获得公平 CPU 时间
├── 交互性 - 交互式任务优先
├── 简单性 - 代码简洁易维护
└── 可扩展性 - 支持多核系统
```

### 虚拟运行时间 (vruntime)

```
vruntime 计算:
vruntime += delta_exec * (NICE_0_LOAD / p->load.weight)

特点:
- 权重越大，vruntime 增长越慢
- 低优先级任务 vruntime 增长快
- 选择 vruntime 最小的任务运行
```

### 红黑树结构

```c
// CFS 运行队列

    struct rb_root tasks_timeline;  // 红黑树
    struct rb_node *rb_leftmost;    // 最左节点 (vruntime 最小)
    u64 min_vruntime;               // 最小 vruntime
    u64 exec_clock;                 // 执行时间
    unsigned int nr_running;        // 运行任务数
    // ... 更多字段
```

### 任务入队

```c
// kernel/sched/fair.c
static void
enqueue_entity(struct cfs_rq *cfs_rq, struct sched_entity *se, int flags)
{
    update_curr(cfs_rq);
    update_stats_enqueue(cfs_rq, se, flags);
    
    if (!se->on_rq)
        __enqueue_entity(cfs_rq, se);
    
    se->on_rq = 1;
}

// 插入红黑树
static void
__enqueue_entity(struct cfs_rq *cfs_rq, struct sched_entity *se)
{
    struct rb_node **link = &cfs_rq->tasks_timeline.rb_node;
    struct rb_node *parent = NULL;
    struct sched_entity *entry;
    s64 key = entity_key(cfs_rq, se);
    
    while (*link) {
        parent = *link;
        entry = rb_entry(parent, struct sched_entity, run_node);
        if (key < entity_key(cfs_rq, entry))
            link = &(*link)->rb_left;
        else
            link = &(*link)->rb_right;
    }
    
    rb_link_node(&se->run_node, parent, link);
    rb_insert_color(&se->run_node, &cfs_rq->tasks_timeline);
}
```

### 任务选择

```c
// 选择下一个任务
static struct sched_entity *pick_next_entity(struct cfs_rq *cfs_rq)
{
    struct sched_entity *se = __pick_first_entity(cfs_rq);
    
    // 选择 vruntime 最小的任务
    if (se)
        update_stats_wait_end(cfs_rq, se);
    
    return se;
}

static inline struct sched_entity *
__pick_first_entity(struct cfs_rq *cfs_rq)
{
    return rb_first_cached(&cfs_rq->tasks_timeline);
}
```

## 🔄 实时调度器

### 调度策略

```
实时调度策略:
├── SCHED_FIFO      // 先进先出
├── SCHED_RR        // 轮转
└── SCHED_DEADLINE  // 截止时间
```

### SCHED_FIFO

```
特点:
- 固定优先级
- 抢占式调度
- 无时间片限制
- 运行到阻塞或自愿让出

使用场景:
- 硬实时任务
- 关键中断处理
- 低延迟要求
```

### SCHED_RR

```
特点:
- 固定优先级
- 时间片轮转
- 同优先级公平

时间片计算:
sched_rr_timeslice = sysctl_sched_rr_timeslice (默认 100ms)
```

### 实时运行队列

```c
struct rt_rq {
    struct rt_prio_array active;    // 优先级数组
    unsigned int nr_running;        // 运行数
    unsigned int nr_pushable_tasks; // 可推送任务
    int highest_prio;               // 最高优先级
    // ... 更多字段
};

// 优先级数组 (100 个 RT 优先级)
struct rt_prio_array {
    DECLARE_BITMAP(bitmap, MAX_RT_PRIO);
    struct list_head queue[MAX_RT_PRIO];
};
```

## ⏰ SCHED_DEADLINE 调度器

### EDF 算法

```
Earliest Deadline First:
- 选择截止时间最早的任务
- 保证实时约束
- 基于带宽预留
```

### 练习 3: 调度延迟测试

```c
// latency_test.c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>
#include <sched.h>

#define TEST_DURATION 10  // 秒
#define INTERVAL_US 100   // 微秒

int main()
{
    struct sched_param param = { .sched_priority = 99 };
    struct timespec start, now;
    long long count = 0, late_count = 0;
    
    // 设置最高实时优先级
    sched_setscheduler(0, SCHED_FIFO, &param);
    
    clock_gettime(CLOCK_MONOTONIC, &start);
    
    while (1) {
        clock_gettime(CLOCK_MONOTONIC, &now);
        
        long long elapsed = (now.tv_sec - start.tv_sec) * 1000000000LL
                          + (now.tv_nsec - start.tv_nsec);
        
        if (elapsed >= TEST_DURATION * 1000000000LL)
            break;
        
        // 检查是否按时唤醒
        long long expected = count * INTERVAL_US * 1000;
        if (elapsed > expected + 50000)  // 50us 容差
            late_count++;
        
        count++;
        usleep(INTERVAL_US);
    }
    
    printf("总迭代：%lld\n", count);
    printf("延迟次数：%lld (%.2f%%)\n", 
           late_count, (double)late_count/count*100);
    
    return 0;
}
```

## ✅ 总结

Linux 进程调度核心要点：

- **CFS** - 完全公平调度器，默认调度策略
- **实时调度** - SCHED_FIFO/RR/DEADLINE
- **vruntime** - 虚拟运行时间保证公平
- **负载均衡** - 多核系统任务分配
- **EAS** - 能效感知调度优化功耗
- **调试工具** - /proc、ftrace、perf

调度器是内核的心脏！
