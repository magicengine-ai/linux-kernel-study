> x86_64 平台完整编译流程

# 基础编译工具
sudo apt-get update

## 🔧 编译步骤

### 1. 获取内核源码

```bash
# 方法 1: 从 kernel.org 下载
wget https://cdn.kernel.org/pub/linux/kernel/v6.x/linux-6.x.tar.xz
tar -xf linux-6.x.tar.xz
cd linux-6.x

# 方法 2: 从 Git 克隆
git clone https://github.com/torvalds/linux.git
cd linux
```

### 2. 清理源码树

```bash
# 清理编译产物
make clean

# 彻底清理 (包括配置)
make mrproper

# 额外清理
make distclean
```

### 3. 配置内核

```bash
# 方法 1: 使用默认配置 (推荐)
make defconfig

# 方法 2: 基于当前运行内核配置
cp /boot/config-$(uname -r) .config
make olddefconfig

# 方法 3: 图形界面配置
make menuconfig

# 方法 4: 最小配置
make tinyconfig
make allnoconfig
# 然后启用必需选项
scripts/config --enable CONFIG_64BIT
scripts/config --enable CONFIG_X86_64
make olddefconfig
```

### 4. 查看配置

```bash
# 查看当前配置
grep "CONFIG_" .config | head -20

# 搜索特定选项
grep "CONFIG_BLK_DEV" .config

# 查看编译目标
make listconfig
```

### 5. 编译内核

```bash
# 编译内核和模块 (使用所有 CPU 核心)
make -j$(nproc)

# 仅编译内核
make bzImage

# 仅编译模块
make modules

# 编译特定架构
make ARCH=x86_64 -j$(nproc)
```

### 6. 安装模块

```bash
# 安装内核模块
sudo make modules_install

# 安装到指定目录
make INSTALL_MOD_PATH=/tmp/mykernel modules_install
```

### 7. 安装内核

```bash
# 安装内核
sudo make install

# 手动安装
sudo cp arch/x86/boot/bzImage /boot/vmlinuz-6.x-custom
sudo cp System.map /boot/System.map-6.x-custom
sudo cp .config /boot/config-6.x-custom

# 更新 initramfs
sudo update-initramfs -c -k 6.x-custom

# 更新 GRUB
sudo update-grub
```

## 📊 编译时间优化

### 并行编译

```bash
# 使用所有 CPU 核心
make -j$(nproc)

# 指定核心数
make -j8
make -j16
```

### 使用 ccache

```bash
# 安装 ccache
sudo apt-get install ccache

# 配置
export CC="ccache gcc"
export CXX="ccache g++"

# 查看统计
ccache -s
```

### 使用 tmpfs

```bash
# 创建 tmpfs 用于编译
sudo mount -t tmpfs -o size=10G none /tmp/build
cd /tmp/build
make O=/tmp/build -j$(nproc)
```

## 🔍 常见问题

### flex/bison 缺失

```bash
# 错误信息
/bin/sh: 1: flex: not found
/bin/sh: 1: bison: not found

# 解决方案
sudo apt-get install flex bison
```

### SSL 头文件缺失

```bash
# 错误信息
fatal error: openssl/ssl.h: No such file or directory

# 解决方案
sudo apt-get install libssl-dev
```

### 内存不足

```bash
# 错误信息
Killed
make: *** [Makefile:xxx] Error 137

# 解决方案
# 1. 减少并行编译核心数
make -j2

# 2. 增加 swap
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 磁盘空间不足

```bash
# 检查空间
df -h

# 清理
make clean
make mrproper

# 编译到外部磁盘
make O=/mnt/external/build -j$(nproc)
```

## 📝 编译输出

### 重要文件

| 文件 | 位置 | 说明 |
|------|------|------|
| bzImage | arch/x86/boot/bzImage | 压缩内核镜像 |
| vmlinux | ./vmlinux | 未压缩内核 (调试用) |
| System.map | ./System.map | 内核符号表 |
| .config | ./.config | 配置文件 |
| 模块 | ./modules/ | 内核模块 |

### 安装后文件

```bash
# /boot/ 目录
/boot/vmlinuz-6.x-custom      # 内核镜像
/boot/initrd.img-6.x-custom   # 初始 RAM 磁盘
/boot/System.map-6.x-custom   # 符号表
/boot/config-6.x-custom       # 配置文件

# /lib/modules/ 目录
/lib/modules/6.x-custom/      # 内核模块
/lib/modules/6.x-custom/build # 编译链接
/lib/modules/6.x-custom/source # 源码链接
```

## ✅ 验证编译

### 检查内核版本

```bash
# 查看编译版本
make kernelrelease

# 查看版本信息
cat include/generated/uapi/linux/version.h
```

### 测试启动

```bash
# 1. 更新 GRUB
sudo update-grub

# 2. 重启系统
sudo reboot

# 3. 在 GRUB 菜单选择新内核

# 4. 验证运行内核
uname -r
uname -a
```

### 检查模块

```bash
# 列出模块
ls /lib/modules/$(uname -r)/

# 查看模块依赖
depmod -a

# 测试加载模块
modprobe -n module_name
```

1. **准备环境** - 安装编译工具

5. **安装模块** - make modules_install
6. **安装内核** - make install
7. **验证测试** - 重启验证
